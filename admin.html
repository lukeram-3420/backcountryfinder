How to use this tab

Toggle a provider off to hide all their courses from the site. Click Run to trigger a fresh scrape for that provider. Click Validate to run the validator — use this after fixing a mapping or summary to confirm the flag is resolved without waiting for the next cron run. Last run shows the most recent scraper execution per provider. Use Validate after Run to confirm fixes were applied. Validate alone is useful after fixing summaries or price exceptions — for mapping or scraper fixes, always Run first then Validate. Run all triggers a full scrape of all providers — expect 45-60 minutes. Column headers are clickable to sort ascending or descending.

How to use this tab

Pending mappings are Haiku suggestions waiting for your review. Approve to add permanently to the mapping table, or edit the activity first if Haiku got it wrong. Use Add mapping to insert a mapping directly without waiting for the scraper to surface a pending suggestion — useful for pinning a title to an activity proactively. Approved mappings with 0 courses are safe to delete — they will be re-suggested if a matching course appears in future. Column headers are clickable to sort ascending or descending.

How to use this tab

Pending summaries are Haiku-generated descriptions waiting for approval. Edit the text if needed, then Approve to publish it to the site instantly. Courses with bad_description user flags appear here — approving a corrected summary unhides the course immediately.

How to use this tab

Auto-hidden courses are hidden from the site until the flag is resolved. Use Add mapping to permanently fix activity mismatches — the mapping is saved and applied on every future scrape. Use Diagnose on duplicates to let Haiku determine if it is safe to whitelist or needs a scraper fix. Use Regenerate to fix empty or mismatched summaries. Warnings are courses still visible on the site but with data quality issues — use Mark as expected to permanently suppress price outliers. Click Re-validate all to refresh flags and warnings after making fixes. Flags and warnings auto-refresh after each validate run.

How to use this tab

Track provider onboarding from candidate → scouted → scraper_built → live. Skipped providers are hidden. Click Add provider to paste a URL — Claude Haiku fetches the site and pre-fills name, location, platform, complexity, priority, plus Google Places rating. Click Copy prompt ↗ on any non-live row to copy a Claude Code instruction for building the scraper. Column headers for Name, Platform, Status, Priority are clickable to sort. Click Edit to change status/platform/priority/notes inline. Pipeline rows auto-sync to 'live' whenever the matching provider is active=true in the Providers tab — and revert to 'candidate' if that provider is later toggled off.
