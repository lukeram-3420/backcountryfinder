How to use this tab

Toggle a provider off to hide all their courses from the site. Click Run to trigger a fresh scrape for that provider. Click Validate to run the validator — use this after fixing a mapping or summary to confirm the flag is resolved without waiting for the next cron run. Scraper is Yes/No based on whether we already have scraper coverage for this provider's booking platform. The set is derived live from the distinct booking_platform values across all active providers (auto-grows as new platforms are onboarded — no hardcoded list). A "No" here usually means providers.booking_platform is null / unknown — backfill it by running backfill_platforms.py or via the Pipeline tab's Detect button. Last run shows the most recent scraper execution per provider. A yellow ⚠ drop ↗ badge appears on the status cell when the last scrape returned >30% fewer courses than the previous run — click it to jump to the GitHub Actions workflow for that provider and investigate the scraper run (Initiative 7). Use Validate after Run to confirm fixes were applied. Validate alone is useful after fixing summaries — for mapping or scraper fixes, always Run first then Validate. Run all triggers a full scrape of all providers — expect 45-60 minutes. Column headers are clickable to sort ascending or descending.

How to use this tab

Exception inbox — summaries go live automatically. Only three cases land here: (1) summary bleed — the validator detected identical text on two different course titles and auto-hid the second occurrence, (2) generation failed — the validator's inline backfill could not produce a summary for a course with null text (visible card, blank summary), (3) user bad_description reports — someone flagged the description as bad. Edit the Card description textarea (shown on cards) as needed, then click Save. Save writes the text, clears any auto-flag or user flag on the course, and records an exception so the validator does not re-flag the same summary text on future runs. Click Regenerate for a fresh Haiku pass if you want a different starting point. The Search document is keyword text for Algolia and is not shown to users.

How to use this tab

Auto-hidden courses are hidden from the site until the flag is resolved. Summary-related issues (bleed, user bad_description reports, generation failures) go to the Summary Review tab — this tab surfaces duplicates, date escalations, price escalations, and availability warnings. Duplicates are always scraper bugs (Initiative 6) — each group shows a read-only scraper_<provider_id>.py ↗ link directly to the offending file on GitHub; open it, fix the iteration logic, re-run the scraper, and the duplicates clear on the next validate. Use Clear all only if you need to suppress a title pattern that cannot be fixed provider-side. Date escalations appear 24 hours after a past-date or far-future (>2 yr) course is first flagged — copy the email body, send to the provider, then Clear to suppress that course_id permanently. Price escalations appear 24 hours after a course with a zero or negative price is first detected (same copy-email-then-Clear pattern). Warnings are courses still visible on the site but with availability gaps. Scraper count-drop alerts moved to the Providers tab as a yellow badge on the status cell (Initiative 7). Click Re-validate all to refresh flags and warnings after making fixes.

How to use this tab

Discovery Cloud (top) controls the search terms the weekly refresh_discovery_cloud.py + discover_providers.py cron uses to find new providers. Activity terms come from course-title bigrams across 2+ providers; location terms come from provider locations + location mappings. Add a manual term to search for niches not yet in your data. Click Remove to block a term — it stays in the table greyed out and the refresh script will not re-add it on its next run. Click Restore to un-block. Quality shows "X found / Y skipped" — >80% skip rate (with 5+ total) gets a warning.

Provider pipeline (bottom): providers already live in the Providers tab are auto-hidden. This list shows candidates still to be onboarded. Click Add provider to paste a URL — Claude Haiku fetches the site and pre-fills name, location, platform, complexity, priority, plus Google Places rating. Click the small Detect button in the Platform column to re-run signature detection (fetches the homepage, matches against PLATFORM_SIGNATURES) — useful for rows Haiku guessed wrong. Click Copy prompt ↗ on any row to copy a Claude Code instruction for building the scraper. Column headers for Name, Platform, Status, Priority are clickable to sort. Click Edit to change status/platform/priority/notes inline.
